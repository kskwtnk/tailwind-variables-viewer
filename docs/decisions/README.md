## このドキュメントの目的

- 技術選定の理由を明確に記録
- いつどんな決定をしたかを時系列で把握
- 将来の意思決定の参考資料
- 新規参加者への背景説明

## ファイル命名規則

- フォーマット: `YYYY-MM-DD-short-title.md`
- 例: `2026-01-06-vanilla-js.md`
- 日付は意思決定を行った日

## 更新ルール

- 重要な技術選定時に追記
- 状態: 提案 → 採用/却下/廃止
- 日付順にソート
